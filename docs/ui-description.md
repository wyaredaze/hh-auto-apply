# HH Auto Apply — Основной интерфейс (Side Panel)

## Структура UI

### Заголовок
- Название "HH Auto Apply" по центру
- Иконка настроек (шестерёнка) справа — открывает панель общих настроек

### Панель общих настроек (скрыта по умолчанию)
- Чёрный список (через запятую)
- Авто-действие при отсутствии ответа пользователя (radio):
  - Ждать ответа пользователя (бесконечно)
  - Авто-ответ LLM (с таймаутом, по умолчанию 20 сек)
  - Пропустить вакансию (с таймаутом)

### Загрузка резюме
- Drop-zone для PDF файла (drag & drop или клик)
- Показывает имя загруженного файла
- Зелёная рамка когда файл загружен

### Провайдер
- Выпадающий список: Ollama, LM Studio, OpenAI, Claude
- Иконка настроек провайдера (шестерёнка) — открывает панель:
  - URL и порт (для Ollama/LM Studio)
  - Выбор модели (dropdown, загружается с API провайдера)
  - Кнопка обновления списка моделей
  - API Key (для OpenAI/Claude)

### Управление
- Кнопка "Запустить" (фиолетовая) / "Остановить" (красная)
- Счётчик: Отправлено X / Всего Y

### Лог
- Монопространственный блок с цветовой разметкой:
  - Зелёный — успех
  - Красный — ошибка
  - Оранжевый — предупреждение
  - Серый — информация

### Секция вопроса (появляется при паузе)
- Пульсирующая рамка (фиолетовый ↔ оранжевый)
- Название вакансии, номер вопроса
- Текст вопроса
- Блок "Ответ LLM:" / "Найден ответ из базы:" (если есть)
- Для radio/checkbox — варианты с подсветкой рекомендации LLM
- Для текстовых — textarea с авто-высотой, кнопка очистки (×)
- Кнопки: "Отправить мой", "Из базы" (если из базы), "Отправить LLM"
- Кнопка "Пропустить вакансию" (отдельная строка, прозрачная)

### База ответов (details/summary, свёрнута)
- Список сохранённых Q&A пар
- Кнопка удаления (×) у каждой записи
- Кнопки: "Дедупликация" (оранжевая), "Очистить всё" (красная)

### Debug: вопросы работодателей (details/summary, свёрнута)
- Лог всех вопросов и ответов по вакансиям
